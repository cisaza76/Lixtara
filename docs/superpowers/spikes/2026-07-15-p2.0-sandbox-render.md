## 5. Issues found (fixable harness/env, not architectural)

1. **`xz` missing in the base** → the static ffmpeg `.tar.xz` couldn't unpack (`tar: xz: Cannot
   exec`), so `ffprobe` wasn't installed and QA failed (exit 127). **Fix:** `dnf install -y xz tar`
   before extracting (applied → all ffprobe checks then passed). In production, put ffmpeg/ffprobe
   in the prebuilt base.
2. **Opaque `runCommand` 400 for a missing binary** — calling `sandbox.runCommand("ffprobe", …)`
   when ffprobe is absent throws `Status code 400` (empty body) via the SDK's auto-resume, instead
   of a clean exit code. **Lesson:** run in-sandbox commands through `sh -c "…"` so a missing binary
   returns a clean **exit 127** you can handle. (This is what caused the two earlier aborted runs.)
3. Chromium shared libraries must be pre-installed (Amazon Linux 2023 default image lacks them) —
   use Remotion's linux-deps list.

- QA via ffprobe works and gives exactly the fields the plan's QA needs (codec/res/fps/duration).
